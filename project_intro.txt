# MESClient

MES（製造執行系統）Client 端，前後端同在一個 Web 專案中。

## 技術棧

  ┌──────────────┬──────────────────────────────────────────────┐
  │ 項目         │ 內容                                         │
  ├──────────────┼──────────────────────────────────────────────┤
  │ 框架         │ ASP.NET Core (.NET 9)                        │
  ├──────────────┼──────────────────────────────────────────────┤
  │ 架構         │ MVC（Razor Views）+ API Controllers          │
  ├──────────────┼──────────────────────────────────────────────┤
  │ ORM          │ Entity Framework Core 9 + SQL Server         │
  ├──────────────┼──────────────────────────────────────────────┤
  │ 前端         │ Kendo UI（MVVM）+ Bootstrap + jQuery + ES6   │
  ├──────────────┼──────────────────────────────────────────────┤
  │ DI/AOP       │ AspectCore（動態代理）+ 自動註冊(AddAutoDi)  │
  ├──────────────┼──────────────────────────────────────────────┤
  │ 認證/授權    │ JWT + 自訂 PermissionAuthorizationHandler    │
  ├──────────────┼──────────────────────────────────────────────┤
  │ 安全         │ AntiXss Middleware + RateLimiting + CSP      │
  ├──────────────┼──────────────────────────────────────────────┤
  │ 排程         │ Hangfire（SQL Server 儲存）                  │
  ├──────────────┼──────────────────────────────────────────────┤
  │ 快取         │ Redis (StackExchange) / 可退回 MemoryCache   │
  ├──────────────┼──────────────────────────────────────────────┤
  │ 錯誤監控     │ Elmah + Elastic APM（二擇一，互斥）          │
  ├──────────────┼──────────────────────────────────────────────┤
  │ API 文件     │ Swagger (Swashbuckle)                        │
  ├──────────────┼──────────────────────────────────────────────┤
  │ 日誌         │ NLog（寫入DB: SystemLog / EventLog /         │
  │              │   TrackTransaction） + JSNLog（前端→後端）   │
  ├──────────────┼──────────────────────────────────────────────┤
  │ 多語系       │ .resx 資源檔 + CultureMiddleware             │
  ├──────────────┼──────────────────────────────────────────────┤
  │ 程式碼生成   │ T4 模板（Enum/Resource → JS 同步）           │
  ├──────────────┼──────────────────────────────────────────────┤
  │ 前端打包     │ Gulp（ES Module import 路徑加版本戳）        │

  └──────────────┴──────────────────────────────────────────────┘

## Solution 分層結構（4 層 + 測試）

  src/
  ├── Layer1（共用基礎）
  │   ├── LC.Core/        — 共用工具、Enums、Filters、Middlewares、認證
  │   └── Resource/       — .resx 多語系資源檔

  │   ├── Domain/         — 實體(Entities)、Enums、ViewModels、AOP、Middleware

  │   └── LCApiLogics/    — LC 平台 API 邏輯
  │
  ├── Layer3（業務邏輯）
  │   ├── MenuLogics/     — 選單邏輯
  │   ├── ModuleLogics/   — 模組樹解析
  │   ├── ComboLogics/    — 下拉選單邏輯
  │   ├── CommonLogics/   — 通用業務邏輯
  │   ├── FileLogics/     — 檔案處理邏輯
  │   └── LCERPApiLogics/ — ERP API 整合邏輯
  │
  ├── Layer4（服務層）
  │   ├── AuthServices/       — 使用者/角色/選單 服務 + EventAOP
  │   ├── SSServices/         — 系統設定/事件日誌 服務
  │   ├── FileServices/       — 檔案服務
  │   ├── LCERPApiServices/   — ERP API 服務（HttpClient 封裝）
  │   └── ERPBoardServices/   — ERP 看板服務
  │
  └── WebSite（主 Web 應用）
      ├── Controllers/        — MVC Controllers（18 個，回傳 View）
      ├── ApiControllers/     — API Controllers（19 個，回傳 JSON）
      ├── Views/              — Razor Views（16 個功能模組）
      ├── Filters/            — 全域過濾器（Exception / Auth）
      ├── Job/                — Hangfire 排程任務
      ├── wwwroot/js/         — 前端 JS（ES6 模組化 + Kendo 擴充）

  test/
  ├── ServiceLayer.Tests/     — 服務層測試
  └── AuthServices.Tests/     — 認證服務測試

## 主要功能模組

  製程報工(ManufactureProcess)、派工看板(DispatchBoard)、
  排程看板(ScheduleBoard)、工單看板(TaskBoard)、派工總覽(DispatchOverview)、
  工單總覽(TaskOverview)、檢驗時效(InspectionTiming)、
  綜合報表(CombinedReporting)、攝影機(Camera)、
  使用者/角色管理(User/Role)、系統設定(SysOption)、事件日誌(EventLog)

## 開發重點注意

  - DI 自動註冊：以 "Logic" / "Service" 結尾的類別自動掃描注入
  - 前端 JS 改動後，發佈時需跑 `npx gulp release` 加版本戳
  - Elmah 與 ElasticAPM 互斥，由 appsettings 的 ElasticApm:Enabled 控制
  - Redis 連線可選，未設定時退回 MemoryCache
  - JWT 金鑰持久化到 DB（DataProtection PersistKeysToDbContext）

## 5 層架構慣例（重要）

  1. View       — Razor Views + Kendo UI MVVM 綁定
  2. Controller — 路由、參數解析（方法體 3~5 行）
  3. Service    — 業務邏輯（商業規則在此層）
  4. Logic      — 資料查詢/組裝
  5. Domain/DbContext — EF Core 實體與資料庫

  ⚠ Controller 禁止包含業務邏輯，只做：參數驗證 → 調用 Service → 返回結果

## 開發重點注意

  - DI 自動註冊：以 "Logic" / "Service" 結尾的類別自動掃描注入
  - EventAOP 自動註冊：AddEventAOPAutoDi("Service") 自動掃描 AOP 攔截
  - 多語系流程：修改 translations.csv → 執行 Exec-TT-Resources.ps1 → 生成 .resx + Resources.js
  - Enum 同步：修改 C# Enum → 執行 Exec-TT-Domain.ps1 → 生成 Domain.js
  - SCSS 編譯：後台 `_Design/gulp build-css`，前台 `_Design_Front/gulp build-css`
  - 新功能開發四步驟：ApiGuid.cs → MenuAttr → AuthFilter → MenuLogics（不可省略）
  - 強制 TDD：新增 Service 方法前先寫測試
  - 開發治理：遵循 .specify/memory/constitution*.md 憲法文件
  - 功能規格：specs/ 目錄存放各功能的需求規格

## 核心業務流程（秤重作業）

  1. Kiosk 登入 → 透過 ERP API KioskLogin 取得 Token
  2. 查詢暫收單（掃描條碼 or 依日期查詢）
  3. 選擇暫收單 → 進入分級秤重介面
  4. 選擇作物 → 選擇品級(A/B+/B/C/D) → 選擇規格
  5. 秤重（觸控數字鍵盤輸入）→ 選擇箱籃/棧板 → 新增明細
  6. 列印秤重明細
  7. 送出分級結果 / 新增進貨退出單

## ERP API 串接項目（LCERPApiLogics → ERP PMGradedPurchase API）

  - Login / Logout（Kiosk 認證）
  - GetDocumentsByDate / GetDocumentsByCode（暫收單查詢）
  - GetCrops / GetGradedProductLevels / GetGradedProductSpecs（作物/品級/規格）
  - GetMetarialsByType（箱籃/棧板）
  - GetWarehouses（倉庫）
  - GetDocumentDetails / AddDocumentDetail / DelDocumentDetails（秤重明細 CRUD）
  - CommitDocument（送出分級結果）
  - CreatePurchaseReturn（進貨退出）

  【秤重作業】Weighing — 核心功能，暫收單分級秤重（觸控介面 + 數字鍵盤）
  【事件日誌】EventLog — 系統操作紀錄
  【首頁】    Home — Dashboard / 登入
  【系統管理】共用帳號/權限管理（透過 AuthServices）

## 架構特色（與 MESClient/CSS 共通模式）

  - 4 層分層架構（Layer1~4），專案命名慣例相同

  - ComboManage：統一管理下拉選單資料源

  - KioskHeaderMiddleware：辨識 Kiosk 裝置的專用 Middleware
  - 本系統功能精簡，核心僅一個業務模組（秤重），其餘皆為共用基礎設施

  - Migration.sql 會在部署時複製到輸出目錄
